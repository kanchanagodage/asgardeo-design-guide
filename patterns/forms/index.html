    Forms

    When to use

    Forms are incredibly common in user interfaces and their design and usage continues to evolve as input methods get smarter and more and more people use mobile and tablet devices. You might design a form for a user to

    • Sign up for / log into an account
    • Register application
    • Add users
    • Add user groups
    • Configure settings, (e.g. application settings)
    • Take a survey
    • Provide feedback

    Formatting

    All forms are comprised of six elements:

    1. Labels: Inform users what the corresponding input fields mean.
    2. Input fields: Enable users to provide information. Information can be entered through a variety of different input fields ranging from text fields, checkboxes, and many other types.
    3. Helper text: Provides assistance on how to fill out a field. Helper text is optional.
    4. Placeholder text: Hints at what goes into a field. Placeholder text is optional.
    5. Actions: Allow users to submit a form.
    6. Validation: Ensures the data submitted by the user conforms to acceptable parameters.

    Form logic

    Single selection

    Radio buttons are used when there is a list of two or more options that are mutually exclusive and the user must select exactly one choice. In other words, clicking a non-selected radio button will deselect whatever other button was previously selected in the list.

    Use Radio buttons

    Use a Listbox

    If the user knows what they are looking for, use a Listbox with a form field and let them type and filter. Use a Dropdown if the space is limited. (e.g., Select Country, City)

    Multi-selection

    Checkboxes are used when there are lists of options and the user may select any number of choices, including zero, one, or several. In other words, each checkbox is independent of all other checkboxes in the list, so checking one box doesn’t uncheck the others. A stand-alone checkbox or a toggle can be used for a single option that the user can turn on or off.

    Use Checkboxes

    Use a Listbox

    If the user knows what they are looking for, use a Listbox with a form field and let them type and filter. Use a Dropdown if the space is limited.

    Single selection

    Select elements are used for fields in which a single selection is required and there are a large number of possible options.

    Use a single check box if you have to save the action by clicking a button

    Use a toggle if the action can save itself (Without clicking the button)

    Validation and errors

    Form validation

    We recommend validating the users’ data before form submission. Use visual cues to guide the user as to where the problem lies within the form. This will help to easily identify the elements that need to be corrected.

    The validation should appear when the user has clicked away from the text field. Once the user corrects the errors within the text field, the validation should disappear once the data is rendered as valid.

    The validation label below the field should be as informative as possible when describing the issue with the users data. For example, if password limitations require 16 characters, but the user inputs a password with only six characters, the text should read something like, “Password must be at least 16 characters.”

    Optional versus required fields

    Add red "*" to the end of the labels for Required fields except for the Login form. All other fields are optional
